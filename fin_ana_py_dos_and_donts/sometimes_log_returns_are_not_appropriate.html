Do not use log-returns for long-term performance analysis¶

(Alexander Klos and Matthias Zuckschwert)

The purpose of this notebook is to explain why compounding log returns may be misleading when analyzing an asset's long-term performance.

The log is an approximation. This approximation is quite good near zero, but deteriorates for values that are not near zero. Thus, when using log returns, there is an approximation error that is larger for more extreme returns. If you use log returns in an analysis of the long-term performance of an asset, you would compound all the small errors caused by the logarithm. Let's look at an example using the value-weighted market returns from Ken French's data library.

In [3]:
ds = web.DataReader('F-F_Research_Data_Factors', 'famafrench', start='1927-01-01', end='2020-12-31')[0]
mp = (ds['Mkt-RF']+ds['RF'])/100

Compute the logarithmic approximation for the market portfolio returns.

Note that the log-return is always smaller than the actually returns. You are underestimating the performance.

How would an investment of $P_0$ currency units perform using these returns? Recall that the gross return is defined as

$1+R_t = \dfrac{P_t}{P_{t-1}}$.

Hence, the price at time $t$ is

$P_t = P_{t-1}(1+R_t)$.

Substituting for $P_{t-1}$ and iterating yields that the price at time $t$ can be rewritten to

$P_t = P_0 \cdot\prod\limits_{i=1}^{t}(1+R_i)$,

where $P_0$ represents the initial investment.

In the following plot the performance of the investment of $P_0 = 1$ is computed and plotted with gross- and log-returns.

Conclusion: Do not use log-returns for long-term performance analysis.
